# AI 技术周报示例

## 示例 1：OpenAI 发布重大更新

输入：
- 周一：OpenAI 发布 Operator，可以自动操作浏览器
- 周二：Browser Use 项目 GitHub star 突破 10k
- 周三：某开发者分享用 Operator 自动订机票的 demo
- 周四：Anthropic 发布 Claude 3.7，支持 extended thinking
- 周五：国内某大厂跟进发布类似功能

输出：

```markdown
# 【AI技术周刊】第 12 期 | 2025年2月3日

> 本周一句话：AI 从"回答问题"进化到"帮你办事"，Agent 赛道正式开卷。

---

## 🔥 本周头条（2条）

### 1. OpenAI 发布 Operator：AI 开始帮你操作浏览器

**发生了什么：**
OpenAI 正式发布 Operator，一个能自动操作浏览器完成任务的 AI Agent。它可以看到网页、点击按钮、填写表单，像真人一样浏览网页并完成任务。目前仅对 Pro 用户开放。

**技术影响：**
这是 2025 年 AI 应用方向最确定的趋势——从 Chatbot 到 Agent。对开发者意味着：
- 浏览器自动化将成为标配技能，建议学习 Playwright/Selenium
- 多模态（视觉+操作）能力会成为模型竞争的新维度
- 相关开源项目（如 Browser Use）热度会进一步上升

---

### 2. Anthropic 发布 Claude 3.7 Sonnet：推理能力大幅增强

**发生了什么：**
Claude 3.7 引入 "extended thinking" 模式，可以花更多时间深度推理后再回答。在数学、代码、逻辑推理任务上表现显著提升。

**技术影响：**
推理能力的提升会直接利好 Agent 场景——AI 在操作前能更好地"思考"步骤。如果 Claude 3.7 的 API 稳定，可能会成为构建复杂 Agent 的首选模型。

## 写作技巧总结

### 1. 头条怎么写

**结构：**
- 发生了什么（2-3 句话，说清楚核心事实）
- 技术影响（分层阐述：对创业者/开发者/行业分别意味着什么）

**技巧：**
- 用具体数字增加可信度（价格、性能指标）
- 给出行动建议（"建议关注..."、"可以尝试..."）

### 2. 工具怎么选

**优先级的信号：**
- GitHub Star 数近期快速增长
- 有实际 demo 或详细文档
- 解决的是普遍痛点
- 技术栈流行（Python/TypeScript）

**避免：**
- 只有 README 没有代码的项目
- 过于垂直的工具（除非特别创新）
- 重复造轮子的项目（除非有显著改进）

### 3. 主编观察写什么

**好角度：**
- 观察到的小模式（"这周有 3 个项目在做同一件事..."）
- 对趋势的判断（"接下来可能会..."）
- 一个反常识的洞察

**避免：**
- 泛泛而谈（"AI 发展很快"）
- 纯情绪输出（"太震惊了"）
- 过长的个人故事

### 4. 整体节奏

- 头条：重，需要思考
- 工具：快，直接可用
- 资源：轻，按需取用
- 观察：短，留点印象

让读者可以用 5 分钟快速浏览，也可以花 20 分钟深入阅读。
