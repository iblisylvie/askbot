# 幻灯片布局选择指南

---

## 布局类型详解

### `cover` — 封面（始终用于第 1 张）

**结构：** 日期/来源标签 → 大标题 → 分割线 → 5条以内摘要列表 → 底部数字锚点

**适用：** 固定用于第 1 张，汇总全文核心主题

**示例：**
```
[日期标签]
"今天 AI 圈 · 5件大事"
─────────────────
🏭 主题一：一句话摘要
🦞 主题二：一句话摘要
⚙️ 主题三：一句话摘要
[底部亮眼数字]
```

---

### `numbers` — 数字炸弹

**适用信号：** 有 2-4 个关键数字/数据点，且数字是理解内容的核心

**结构：** 标签 → 标题 → 2x2 数字卡片网格 → 底部补充说明框

**卡片结构：**
- 大数字（32-48px，accent 色）
- 单位/名称（11px，暗色）
- 补充说明（10px，低不透明度）

**例：** Terafab（200亿美元 / 1太瓦 / 2% / 2028年）

---

### `timeline` — 时间线/流程

**适用信号：** 有序事件序列，或有"反转"、"步骤"感的故事

**结构：** 标签 → 标题 → 竖向时间线（圆形序号 → 连接线 → 文字块）

**每个节点：**
- 圆形序号（accent 色背景）
- 粗体事件标题
- 细节补充（低透明度小字）

**例：** Cursor 套壳 Kimi 事件（发布 → 被逆向 → 联创承认 → 额外消息）

---

### `comparison` — 对比表

**适用信号：** 2-5 个对象，每个对象有相似属性可横向对比

**结构：** 标签 → 标题 → 副标题 → 对象卡片列表 → 底部注意事项

**每张卡片：**
- emoji + 名称（粗体，light 色）
- 所属方（小字，低透明度）
- 右侧状态标签（可选，用 accent 色背景）

**例：** 三只龙虾接入微信（StepClaw / AutoClaw / Kimi Claw）

---

### `list` — 要点列表

**适用信号：** 3-6 条独立要点，无明确顺序，每条有标题+补充说明

**结构：** 标签 → 标题 → 条目列表（emoji + 粗体标题 + 细节）

**每条格式：**
```
[emoji]  [粗体标题]
         [细节说明，低透明度，小字]
```

背景：`rgba(255,255,255,0.05)`（暗底）或 `rgba(26,51,40,0.05)`（浅底）

**例：** AI冲击就业（腾讯AI Lab / 网易辟谣 / 百度 / 宇树 / 硅谷）

---

### `mixed` — 数字 + 说明混合

**适用信号：** 有 1-2 个核心数字，但还需要配合文字说明才能讲清楚

**结构：** 标签 → 标题 → 数字对比行（2格） → 详情框（key-value 列表）→ 引用块

**例：** Agent 造 CPU（219字/12小时 → VerCore CPU 规格 → "首个由 Agent 完整构建的 CPU"）

---

### `quote` — 结尾引用（始终用于最后 1 张）

**结构（居中）：** 分割线 → 小标题"今日最值得记住的" → 大号引用文字 → 分割线 → 关键词标签组 → 日期

**背景：** 固定使用深黑底 `#0F1F18`

---

## 布局选择决策树

```
这张幻灯片的内容是...

有 2-4 个具体数字，数字是核心？
  → numbers

是一个有序的事件/步骤/反转故事？
  → timeline

是多个对象横向对比？
  → comparison

是 3-6 条独立要点，无数字？
  → list

有 1-2 个数字 + 需要详细说明？
  → mixed

是结尾卡？
  → quote
```
