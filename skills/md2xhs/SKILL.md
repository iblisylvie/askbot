# md2xhs — Markdown 转小红书轮播图

## 工作流程

### Step 0：加载品牌配置

按优先级检查 EXTEND.md：
```bash
test -f .agents/skills/md2xhs/EXTEND.md && echo "project"
test -f "$HOME/.agents/skills/md2xhs/EXTEND.md" && echo "user"
```

找到则读取；未找到则使用默认品牌色（见下方默认值）。

**EXTEND.md 支持的字段：**
```yaml
brand:
  primary: "#1A3328"    # 暗底背景色
  light: "#F2EDE3"      # 浅底背景色
  accent: "#C44536"     # 强调色（数字、标签、分割线）
  name: "01fish"        # 左上角品牌名（空字符串则不显示）
max_slides: 8           # 最多幻灯片数（默认 8，含封面+结尾）
```

**默认值（无 EXTEND.md 时）：**
- primary: `#1A3328`，light: `#F2EDE3`，accent: `#C44536`
- brand name: 空（不显示）
- max_slides: 8

### Step 1：读取并分析内容

读取输入的 .md 文件，提取：
- 顶层主题列表（H2 / 段落块 / 列表组）
- 每个主题的关键数字、对比项、事件列表
- 文章整体的「最值得记住的一句话」（用于结尾卡）

**幻灯片数量决策规则：**
- 主题数 ≤ 4：每个主题 1 张 + 封面 + 结尾 = N+2 张
- 主题数 5-6：1:1 对应，共 7-8 张
- 主题数 ≥ 7：次要主题合并到一张「速览」卡，控制在 max_slides 内
- 始终以封面（第 1 张）开头，结尾卡（最后 1 张）结束

### Step 2：为每张幻灯片选择布局

读取 `references/layout-guide.md` 了解各布局的使用时机，为每张幻灯片选定一种布局类型：

| 布局 | 适用信号 |
|------|---------|
| `numbers` | 有 2-4 个关键数字/数据点 |
| `timeline` | 事件序列、步骤流程、反转故事 |
| `comparison` | 多个对象并排对比 |
| `list` | 5 条以内的独立要点 |
| `mixed` | 数字 + 说明混合 |
| `quote` | 仅用于结尾卡 |

暗底/浅底**严格交替**：第 1 张（封面）暗底，第 2 张浅底，第 3 张暗底……最后一张（结尾）深黑底（`#0F1F18`）。

### Step 3：生成 HTML

读取 `references/html-components.md` 获取完整的 CSS 框架、视觉组件库和下载脚本。

**生成原则：**
- 复用 html-components.md 的 CSS 基础框架，只替换内容
- 每张幻灯片用 `<div class="slide">` 包裹
- 标题行字号 24-28px，正文 11-14px，大数字 32-48px
- 信息密度适中：每张不超过 6 个信息点，留白充足
- 数字用 accent 色高亮，正文用常规色

**输出文件路径：** `[输入文件目录]/[输入文件名]-小红书版.html`

例：输入 `output/2026-03-22/daily-summary.md` → 输出 `output/2026-03-22/daily-summary-小红书版.html`

### Step 4：告知用户

生成完成后输出：
```
✓ 已生成：[输出路径]
  共 N 张幻灯片：[封面标题] · [各张标题简述]
  浏览器打开后点击「全部下载 ZIP」即可导出 PNG
```

---

## 注意事项

- 严格基于原文内容，不捏造数据或观点
- 如果原文没有数字，不要强行加数字卡片，改用 list 或 quote 布局
- 标题要简洁有冲击力，控制在 12 字以内
- 结尾卡从原文提取一句最有价值的话，不自行发挥
