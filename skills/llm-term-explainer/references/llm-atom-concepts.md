# LLM 原子概念速查表

用于保证解释口径稳定。先给白话，再给技术定义，再给易混淆点。

## Token
- 白话：模型处理文本时切出来的“最小计算块”。
- 技术：Tokenizer 把文本映射为离散 token ID 序列。
- 易混淆：Token 不等于“词”；一个词可能对应多个 token。

## Embedding
- 白话：给每个 token 一个语义坐标。
- 技术：把离散 ID 映射到连续向量空间。
- 易混淆：Embedding 不是解释文本的“词典释义”。

## Positional Encoding
- 白话：告诉模型“谁在前，谁在后”。
- 技术：向 token 表示注入位置信息，弥补自注意力的置换不变性。
- 易混淆：位置编码不是语法规则本身。

## Attention
- 白话：读一个词时，动态决定该参考上下文里哪些词。
- 技术：通过 Query-Key 相似度对 Value 加权求和。
- 易混淆：不是只“盯一个词”，而是分布式加权。

## MHA（Multi-Head Attention）
- 白话：让多组“阅读视角”并行看同一句话，再合并结果。
- 技术：在不同投影子空间并行执行 attention，拼接后线性变换。
- 易混淆：不是简单重复；不同 head 学的是不同关系模式。

## FFN / MLP
- 白话：每个位置上的“信息加工车间”。
- 技术：逐位置的前馈非线性变换层。
- 易混淆：不负责跨位置对齐，跨位置主要由 attention 完成。

## Residual Connection
- 白话：给信息留“快速回路”，防止中途失真。
- 技术：层输入与层输出相加，改善梯度传播和训练稳定性。
- 易混淆：不是跳过学习，而是让学习更容易。

## LayerNorm
- 白话：每一层都先“校准尺度”，避免数值过飘。
- 技术：对特征做归一化并学习缩放、偏移参数。
- 易混淆：与 BatchNorm 不同，不依赖 batch 统计。

## Logits
- 白话：模型给每个候选词的“原始打分”。
- 技术：softmax 前的未归一化分数向量。
- 易混淆：logits 不是概率，softmax 后才是概率分布。

## Temperature
- 白话：控制输出“保守”还是“发散”。
- 技术：softmax 前用温度缩放 logits。
- 易混淆：温度改变采样分布，不改变模型参数。

## KV Cache
- 白话：把前文计算结果缓存起来，避免每次重算。
- 技术：推理时缓存历史 token 的 Key/Value，提高自回归生成效率。
- 易混淆：缓存提升速度，不提升事实正确率。

## RAG
- 白话：答题前先查资料，再组织答案。
- 技术：检索外部知识并拼接到提示词，增强生成。
- 易混淆：RAG 不等于微调；不改模型参数。

## Fine-tuning
- 白话：让模型在特定领域“再训练”。
- 技术：在新数据上继续优化部分或全部参数。
- 易混淆：微调和提示工程、RAG 是不同层面的改造。
